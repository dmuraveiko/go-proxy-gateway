# Открытые вопросы

Блокирующих архитектурных вопросов после ответов от 16 июля 2026 года не осталось.
Зафиксировано:

- один логический `proxy_id` использует одну PostgreSQL; штатно запускается один
  процесс, несколько экземпляров остаются опциональным HA-режимом;
- клиент получает готовое PostgreSQL-хранилище с префиксом `natsproxyclient_`;
- исходящий API реализуется через `http.RoundTripper`;
- точные bytes body и исходная строка URL сохраняются, порядок headers не обещается.

Перед production остаётся подобрать эксплуатационные значения: размеры сообщений,
HTTP timeout, backoff, retention, host limits, NATS ACL, ключи, PostgreSQL HA/PITR и
SLO. Они не меняют архитектуру.
